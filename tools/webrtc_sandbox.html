How to Establish a Secure Link

This utility utilizes WebRTC Data Channels to forge a direct, end-to-end encrypted connection between two browsers. Because SilenVault operates without backend signaling servers, connection keys must be exchanged manually.

01. Generate

The Host clicks "Generate Link Key", copies the cryptographic string, and sends it to the Guest via a secure messenger.

02. Respond

The Guest pastes the Host Key, generates a Return Key, and sends it back to the Host.

03. Establish

The Host pastes the Return Key. The browsers negotiate network traversal and establish a direct lock.

Technical Limitations

  • Volatile Memory: If either party refreshes the page, closes the tab, or loses internet connection, the P2P link is permanently severed. All chat history is instantly wiped from memory.
  • File Size Constraints: While the engine utilizes binary chunking and network backpressure, passing files larger than 500MB may cause browser memory overflow depending on your hardware.
  • Network Firewalls: Highly restrictive corporate firewalls or strict NATs may block direct P2P routing without a TURN relay server.

Security & Privacy

All data transferred through this channel is secured using mandated DTLS (Datagram Transport Layer Security). Because there is no intermediary server processing your data, man-in-the-middle attacks by the host provider (SilenVault) are physically impossible.

Warning

While the transmission pipeline is perfectly secure, the endpoints are human. Never transmit unencrypted crypto seed phrases, SSNs, or master passwords. The peer on the other side can take screenshots, and compromised browser extensions can still read DOM output.
