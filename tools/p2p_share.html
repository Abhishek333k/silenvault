How to Establish a Secure Link

This utility utilizes WebRTC Data Channels to forge a direct, end-to-end encrypted connection between two browsers. Because SilenVault operates with a strict zero-server policy, there are no backend signaling databases. Connection keys must be exchanged manually.

01. Generate

The Host clicks "Create Link Key", copies the resulting cryptographic string, and securely sends it to the Guest (via WhatsApp, Signal, etc).

02. Respond

The Guest pastes the Host's Key into their panel, clicks "Generate Return Key", and sends the new string back to the Host.

03. Establish

The Host pastes the Return Key. The browsers negotiate network traversal, bypass servers entirely, and establish a direct P2P lock.

Technical Limitations

  • ×
    Volatile Memory: If either peer refreshes the page, closes the tab, or loses internet, the connection is permanently severed. All chat history is instantly wiped from RAM.
  • ×
    File Size Constraints: The engine utilizes binary chunking and network backpressure. However, transferring files larger than 500MB may cause browser memory limits to trigger, depending on device hardware.
  • ×
    Symmetric NATs: Highly restrictive corporate firewalls or symmetric NAT networks may block direct P2P routing without a TURN relay server (which we do not use, to ensure privacy).

Security & Privacy Standards

All data transferred through this channel is secured using mandated DTLS (Datagram Transport Layer Security) protocols. Because there is no intermediary server processing or relaying your packets, man-in-the-middle attacks by the host provider (SilenVault) are physically impossible. The connection exists purely between the two IP addresses involved.

Security Disclaimer

While the transmission pipeline is perfectly secure, the endpoints are human. Never transmit unencrypted crypto seed phrases, SSNs, or master passwords. The peer on the other side can take screenshots, and compromised browser extensions on either device can still read DOM output.
